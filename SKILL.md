# kdskill：知识博主蒸馏工具包

你是知识博主内容蒸馏的向导。你的任务是引导用户走完从 SRT 字幕到结构化知识库的完整流程，并在过程中同步沉淀可检索、可评分、可改写的钩子和金句资产。

**核心原则**：能自动的你来做，关键决策点停下来等用户确认。

---

## 内部路由（状态检测）

每次被调用时，先问用户一句话判断当前阶段：

> "你现在进行到哪一步了？"
> 
> 0. 第一次用，从头开始
> 1. 已有 SRT 字幕文件
> 2. 已做好内容分类
> 3. 已确认蒸馏模板
> 4. 已完成批量蒸馏，准备汇总
> 5. 想继续上次未完成的任务

根据用户回答路由到对应步骤。如果用户说"继续上次"，优先检查当前目录下是否有 `.kdskill-state.md` 状态文件；如果没有，再检查旧版 `.distill-state.md`，从中恢复进度。

---

## 完整流程

```
准备素材 → 内容分类 → 设计模板 → 批量蒸馏 → 钩子金句资产层 → 综合汇总(三阶段) → 写入检索规则
   ↓           ↓          ↓           ↓              ↓              ↓                ↓
  步骤1       步骤2      步骤3       步骤4         步骤4.5         步骤5-7           步骤8
```

### 步骤 1：准备素材

**自动化**：

1. 询问用户的视频素材在哪里
2. 确认素材格式：SRT 字幕文件（纯文本），如果不是 SRT，告知用户先用导出工具转换
3. 告知用户推荐的目录结构：

```
你的知识库/
└── 目标博主名/
    └── 00原文件/
        ├── 分类1/
        ├── 分类2/
        └── ...
```

**完成后**：更新状态文件 `.kdskill-state.md`，标记步骤 1 完成。

---

### 步骤 2：内容分类

**自动化**：

1. 列出 `00原文件/` 下所有 SRT 文件名
2. 根据文件名关键词，自动建议分类方案（3-10 个主题）
3. 给出分类建议表格，让用户确认或调整

示例输出格式：

| 建议分类 | 匹配到的文件数 | 示例文件 |
|---------|-------------|---------|
| 01学习方法 | 15 条 | "怎么高效读书...", "自学的方法..." |
| 02赚钱创业 | 8 条 | "普通人怎么赚...", "一人公司..." |
| ... | ... | ... |

**确认点** ⚠️：用户确认分类方案后，AI 自动生成文件夹结构并给出移动文件的命令。**用户必须手动确认分类无误后再继续。**

**约束**：
- 分类数量不限，按实际内容决定
- 每文件夹 ≤30 条（超过则拆分子文件夹如 `第1组/`、`第2组/`）
- 唯一硬约束：每批 ≤30 条，确保 AI 上下文不截断

**完成后**：更新状态文件。

---

### 步骤 3：设计蒸馏模板（确认点 ⚠️）

**这是整个流程最重要的质量控制节点。必须先跑通 1-2 条，确认样式后再批量开工。**

**自动化**：

1. 读取与本 SKILL.md 同目录下的 `知识博主-单条视频蒸馏模板.md`
2. 从素材中选 1-2 条有代表性的 SRT 文件
3. 按模板执行蒸馏分析
4. 输出结果，询问用户：

> "这是模板跑通的效果，请检查 7 个栏目是否完整、道法术提炼是否到位，以及开头钩子、金句字段是否足够支持后续检索。确认后模板冻结，后续批量蒸馏不再改动。要调整的话告诉我要改哪里。"

**确认点** ⚠️：用户确认模板样式后，冻结模板。**中途改模板会导致前后格式不统一。**

**钩子金句检查点**：
- 开头钩子必须包含：钩子类型、心理触发器、开头动作、开头风格、原文、可迁移模板、适用场景、风险。
- 金句清单必须包含：金句原文、位置、功能、句式模式、可迁移模板、可用于开头/中段/结尾的位置判断。
- 如果样本中的钩子或金句质量一般，也要标明等级，不要把弱钩子混入主库。

**完成后**：更新状态文件，记录模板已冻结。

---

### 步骤 4：批量蒸馏

**从这一步开始，每个分类子文件夹需要开一个新对话窗。** 本窗口只负责生成开工指令。

**自动化**：

1. 读取同目录下的 `知识博主-批量蒸馏开工指南.md`
2. 根据用户确认的分类结构，为每个子文件夹生成一条开工指令
3. 输出格式：

```
📋 复制以下指令，在新对话窗中发送：

┌─────────────────────────────────────────────┐
│ 窗口 1：01学习方法/阅读方法（39 条）            │
│                                             │
│ 读取 知识博主-批量蒸馏开工指南.md，执行任务。     │
│ 批次：01学习方法/阅读方法                      │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│ 窗口 2：01学习方法/写作输出（12 条）            │
│                                             │
│ 读取 知识博主-批量蒸馏开工指南.md，执行任务。     │
│ 批次：01学习方法/写作输出                      │
└─────────────────────────────────────────────┘

...（共 N 个窗口）
```

**约束**：
- 每个窗口只处理一个子文件夹
- 每批 ≤30 条
- 多个窗口可以并行
- 每个窗口完成后 AI 会报告产出文件数

**完成后**：用户告知所有批次完成 → 验证所有 `.md` 文件已生成 → 更新状态文件。

---

### 步骤 4.5：钩子金句资产层（新增）

**目标**：在已有单条蒸馏 `.md` 基础上，生成独立的钩子库和金句库，让用户写文案时可以按内容意图检索，而不是手工翻 400 条笔记。

**自动化**：

1. 读取同目录下的 `知识博主-钩子金句资产提取指南.md`
2. 先执行审计模式：统计有开头钩子、金句清单、结尾原文的文件数和格式异常数
3. 先抽最多 20 条样本，覆盖不同分类；如果总量不足 20 条，则全部作为样本，生成样本资产文件
4. 用用户的一篇真实文案做检索验证：自动判断适合的心理触发器、开头动作、开头风格，并推荐可改写钩子和金句
5. 用户确认样本和验证结果后，再进入全量模式
6. 全量输出：

```text
hooks.jsonl
golden_lines.jsonl
opening_moves.jsonl
ending_moves.jsonl
pain_points.jsonl
hook_patterns.md
golden_line_patterns.md
adaptation_templates.md
anti_patterns.md
extraction_report.md
```

**确认点** ⚠️：样本模式和真实文案验证通过后，才允许全量抽取。否则先修字段、分类和评分。

**完成后**：更新状态文件，记录钩子金句资产层已生成。

---

### 步骤 5：综合汇总-第一阶段（分类深度汇总）

**在新对话窗中执行。** 本窗口生成开工指令。

读取同目录下的 `知识博主-综合汇总-第一阶段.md`，生成开工指令：

```
📋 在新对话窗中发送：

读取 知识博主-综合汇总-第一阶段.md，执行任务。
```

**产出**：每个分类生成一份 `【分类名】-深度汇总.md`（日常检索入口层）。

**完成后**：验证所有深度汇总文件已生成 → 更新状态文件。

---

### 步骤 6：综合汇总-第二阶段（跨分类综合）

**在新对话窗中执行。**

读取同目录下的 `知识博主-综合汇总-第二阶段.md`，生成开工指令。

**产出**：
- `知识博主-道法术完整版.md`（道法术顶层归并 + 思想演化脉络）
- `知识博主-视频结构完整图谱.md`（钩子/结构/金句/结尾统计规律）
- `知识博主-钩子金句资产索引.md`（按心理触发器、开头动作、金句功能、适配场景建立检索入口）

**完成后**：验证产出 → 更新状态文件。

---

### 步骤 7：综合汇总-第三阶段（策略与行动）

**在新对话窗中执行。**

读取同目录下的 `知识博主-综合汇总-第三阶段.md`，生成开工指令。

**产出**：
- `知识博主-内容策略总览.md`（内容矩阵 + 选题 + 变现）
- `知识博主研究-我的行动指南.md`（个人适配方案）

**完成后**：验证产出 → 更新状态文件。

---

### 步骤 8：写入检索规则

**自动化**（在汇总阶段全部完成后执行）：

1. 在知识库根目录的 `CLAUDE.md` 中追加检索规则
2. 规则模板：

```markdown
## 【博主名】蒸馏知识库检索规则

当用户提问涉及【博主名】时：
1. 优先 Grep 搜索 *-深度汇总.md 文件
2. 匹配后读取对应深度汇总的相关章节
3. 深度汇总信息不足时，按出处文件列追溯到单条蒸馏原文
4. 如果用户在写文案、改开头、找金句，优先读取钩子金句资产库中的 `hooks.jsonl`、`golden_lines.jsonl` 和 `知识博主-钩子金句资产索引.md`
5. 回答时标注出处（文件名 + 分类）
```

**完成后**：输出完成报告：

```
🎉 蒸馏全流程完成！

产出文件清单：
├── 单条蒸馏：XX 条 .md
├── 分类深度汇总：XX 份
├── 钩子金句资产：hooks.jsonl + golden_lines.jsonl + opening_moves.jsonl + ending_moves.jsonl + pain_points.jsonl
├── 跨分类综合：道法术完整版 + 视频结构图谱 + 钩子金句资产索引
├── 策略行动：内容策略总览 + 个人行动指南
└── 检索规则：已写入 CLAUDE.md

现在你可以随时在知识库中提问关于【博主名】的任何问题了。
```

## 常见问题

**Q: 中途断了怎么恢复？**
A: 说"继续蒸馏"，AI 优先读取 `.kdskill-state.md` 恢复进度；旧项目如果只有 `.distill-state.md`，也可以兼容读取。

**Q: 没有 SRT 字幕怎么办？**
A: 视频导出工具（剪映、飞书妙记、Whisper）都能生成 SRT。先准备好 SRT 再开始。

**Q: 可以蒸馏文章/图文吗？**
A: 本工具包专为视频设计。如果是纯文本内容，可以跳过步骤 1 的 SRT 要求，但模板需要调整——告诉 AI 你的素材类型即可。

**Q: 每批为什么不能超过 30 条？**
A: AI 上下文窗口有上限。超过 30 条可能导致尾部文件被截断遗漏。

**Q: 我不想用 9 个分类，也适用吗？**
A: 分类数量和名称完全由你的素材决定。3 个、5 个、12 个都行。工具包里的 9 分类只是示例。

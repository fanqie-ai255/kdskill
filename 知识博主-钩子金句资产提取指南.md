# 知识博主-钩子金句资产提取指南

> 目标：把已经完成的单条蒸馏 `.md` 文件，进一步整理为可检索、可评分、可改写的钩子库和金句库。
> 适用场景：批量蒸馏完成后；或旧知识库已经有「开头钩子 / 金句清单」，但没有独立资产库时。

---

## 开工指令（复制到新对话窗）

```text
【钩子金句资产提取】

1. 读取 知识博主-钩子金句资产提取指南.md，执行任务。
2. 输入目录：【填入单条蒸馏 md 所在的知识库根目录】
3. 输出目录：【默认：输入目录/钩子金句资产库】
4. 模式：【审计 / 样本 / 验证 / 全量 / 索引】
```

---

## 核心原则

1. 不重新蒸馏整条视频，只抽取已经蒸馏出的结构化字段。
2. 如果结构化字段缺失，再回退到 `## 一、原文` 的开头 3-30 秒做判断。
3. 不把所有钩子都当好钩子；弱钩子进入次级或反例库。
4. 不要求用户手填「想要什么风格」。用户给一篇文案后，由 Skill 自动生成文案检索卡，再匹配钩子和金句。
5. 不照搬对标博主话术，只保存来源、结构、触发器和改写模板。

## 评分规则

### 钩子评分

每条钩子按 10 个维度评分，每项 0-5：

```text
独立性
话题清晰
身份锚定
痛点精准
认知冲突
悬念感
可信度
口播友好
主体兑现
可迁移性
```

分层：

```text
S：42-50，主库强资产
A：34-41，主库可用资产
B：26-33，结构库，原句一般但结构可用
C：18-25，次级或反例库
D：0-17，丢弃
```

### 金句评分

每条金句按 8 个维度评分，每项 0-5：

```text
观点清晰
句式有记忆点
口播友好
可独立传播
能服务结构链
不依赖对标博主人设
可迁移到用户内容
无过度套路感
```

分层：

```text
S：34-40，主库强资产
A：28-33，主库可用资产
B：21-27，结构库
C：14-20，次级或反例库
D：0-13，丢弃
```

## 执行流程

### 1. 审计模式

先统计输入目录：

- `.md` 文件总数
- 有 `### 1. 开头钩子` 的文件数
- 有 `### 3. 金句清单` 的文件数
- 有 `结尾原文` 的文件数
- 字段缺失文件
- 格式异常文件

只输出审计结果，不直接全量抽取。

### 2. 样本模式

抽最多 20 篇，尽量覆盖不同分类。如果输入目录里的有效文件不足 20 篇，则全部作为样本，不要为了凑数引入无效文件。

输出：

```text
hooks.sample.jsonl
golden_lines.sample.jsonl
opening_moves.sample.jsonl
ending_moves.sample.jsonl
pain_points.sample.jsonl
extraction_sample_report.md
```

样本报告必须检查：

- 字段是否够用
- `primary_trigger` 是否比旧钩子类型更适合检索
- `opening_move` 是否能指导开头改写
- `tier` 是否把弱钩子降级
- `fit_for_user` 是否能降级强人设、强社群、强承诺表达

### 3. 真实文案验证闸门

样本模式完成后，必须拿用户的一篇真实文案验证检索效果。

验证流程：

```text
用户文案 / SRT / 初稿
→ 生成文案检索卡
→ 从样本钩子中检索候选
→ 输出推荐钩子类型、可改写开头、可用金句
→ 用户确认是否符合直觉
```

文案检索卡格式：

```text
文案主题：
目标观众：
观众痛点：
核心承诺：
已有可信素材：
内容类型：
视频目标：
适合心理触发器：
适合素材类型：
适合开头动作：
适合开头风格：
不适合的钩子类型：
```

只有满足以下条件，才允许进入全量：

```text
用户认可样本分类口径
用户认可真实文案验证结果
没有发现字段缺失
没有发现评分明显过松或过严
不需要重做核心分类体系
```

如果验证失败，不进入全量；先修字段、分类和评分，再重新跑 20 条样本。

### 4. 全量模式

按全部单条蒸馏文件抽取。

全量报告必须包含：

```text
有效文件数
钩子数
金句数
开头动作数
结尾动作数
痛点记录数
分类覆盖
钩子等级分布
金句等级分布
心理触发器分布
格式异常文件
需要人工复核样本
```

### 5. 索引模式

生成 `知识博主-钩子金句资产索引.md`，至少包含：

- 按心理触发器索引
- 按开头动作索引
- 按开头风格索引
- 按金句功能索引
- 按句式模式索引
- 按适配场景索引
- 反例和慎用表达索引

## 淘汰和降级规则

默认降级或丢弃：

- 开头只是寒暄，没有明确抓手
- 必须依赖标题才能听懂
- 一上来把答案说完，没有悬念
- 只有“我跟你分享一个很重要的东西”这类空话
- 太依赖对标博主人设
- 过度内部黑话
- 承诺太大，正文撑不住
- 只适合卖课或社群转化，不适合用户当前阶段
- 不口语，像 AI 写的

---

## 日常检索方式

当用户给出一篇自己的文案时，不要问用户“你想要什么钩子类型”。先读文案，自动生成检索卡，再从资产库匹配：

1. 同主题或相邻主题
2. 同痛点
3. 同心理触发器
4. 同开头动作
5. 同金句功能
6. `fit_for_user` 高、风险低的候选优先

输出时给用户：

- 当前文案适合的开头策略
- 3 个可改写钩子
- 5 条可借鉴金句
- 每条为什么适合
- 哪些表达不要照搬

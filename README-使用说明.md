# 知识博主蒸馏工具包 —— 使用说明

> 这是一套把知识博主的视频内容系统蒸馏为结构化知识库的完整工具。

---

## 使用前必读

### 关于文件路径
本工具包各文件中引用的路径（如 `E:\02 AI\个人知识库\...`）为作者本机示例路径。**使用前请将路径替换为你自己的知识库目录。**

### 关于内容分类
本工具包以 9 大分类为例组织蒸馏流程，但这只是**示例**，不是固定要求。实际使用时，请根据目标博主的视频内容自行分类：
- 分类数量不限（3 个、5 个、10 个均可，视视频内容而定）
- 分类名称根据内容特点自行命名
- 文中出现的示例分类（学习方法、赚钱创业、自媒体创作等）和示例分批规划仅供参考
- 唯一硬约束：每批处理的视频 **≤30 条**，确保 AI 上下文窗口不截断

---

## 这套工具解决什么问题？

把一位博主的所有内容变成你可以随时检索、复盘、拿来就用的个人智库。

## 快速开始（5 分钟看懂流程）

### 第一步：准备素材

把你喜欢的知识博主的视频导出为 SRT 字幕文件 + MP4 文件，放在同一个文件夹。SRT 是纯文本，Claude Code 能直接读取。

### 第二步：内容分类

将视频按主题分组，方便后续分批蒸馏。操作方式：

1. 浏览你的 SRT 文件名列表，归纳出 3~10 个主题方向（如学习方法、赚钱创业、认知思维等，数量根据实际内容定，不限于任何数字）
2. 在知识库中创建对应的文件夹结构，例如：
   ```
   你的知识库/蒸馏目标博主/
   ├── 01学习方法/
   │   ├── 阅读方法/
   │   └── 写作输出/
   ├── 02赚钱创业/
   └── ...
   ```
3. 把 SRT 文件按主题放入对应文件夹。如果一个文件夹超过 30 条，拆成子文件夹（如 `第1组`、`第2组`……）

> **提示**：如果视频量大、手工分类太慢，可以开一个新对话窗，把文件名列表发给 AI，让它帮你分类并给出移动建议。

### 第三步：设计蒸馏模板（先跑通一条，确认样式）

这一步的目标是**把模板定下来**，后面批量蒸馏时不再改动。

1. 打开 `知识博主-单条视频蒸馏模板.md`，阅读模板结构和提示词
2. 从你的素材里挑 1~2 条有代表性的视频，把 SRT 字幕粘贴到模板提示词末尾，发给 AI
3. 检查 AI 输出的 7 个栏目是否完整、道法术提炼是否到位。不满意就迭代调整，直到满意
4. 确认样式后，**冻结模板**——批量蒸馏阶段不再改动，否则前后格式不统一

> **注意**：这一步不要批量跑。先跑通一条，确认质量再往下走。

### 第四步：批量蒸馏（多窗口并行，逐批执行）

模板冻结后，用 `知识博主-批量蒸馏开工指南.md` 来驱动批量蒸馏。

**执行方式**：每个分类子文件夹开一个新对话窗（或 Agent 后台任务），发送：

```
读取 知识博主-批量蒸馏开工指南.md，执行任务。
批次：你的分类名/子文件夹名
```

**要点**：
- 每次只蒸馏**一个子文件夹**，不要一次跨多个文件夹
- 每批 ≤30 条，避免上下文窗口截断
- 一个窗口跑一批，多个窗口可并行
- 每完成一批，AI 会报告产出文件数

直到所有分类的所有子文件夹都蒸馏完毕，`.md` 蒸馏文件全部生成。

### 第五步：综合汇总（三阶段，按顺序执行）

所有分批蒸馏完成后，依次执行三个汇总阶段。**每个阶段对应一个独立文件，不可跳过、不可乱序。**

**阶段一：分类深度汇总**

打开新对话窗，发送：
```
读取 知识博主-综合汇总-第一阶段.md，执行任务。
```
AI 会遍历所有分类，为每个分类生成一份 `【分类名】-深度汇总.md`。这是日后日常提问时 AI 最先检索的入口层。

**阶段二：跨分类综合**

阶段一完成后，打开新对话窗，发送：
```
读取 知识博主-综合汇总-第二阶段.md，执行任务。
```
AI 会跨所有分类提取共性规律，产出两份文件：
- `知识博主-道法术完整版.md`（道法术全部要点 + 思想演化脉络 + 一句话总结）
- `知识博主-视频结构完整图谱.md`（钩子/结构/金句/结尾的完整模式库）

**阶段三：策略与行动**

阶段二完成后，打开新对话窗，发送：
```
读取 知识博主-综合汇总-第三阶段.md，执行任务。
```
AI 会基于前两阶段的产出，生成两份落地文件：
- `知识博主-内容策略总览.md`（内容矩阵 + 选题策略 + 进化史 + 变现路径）
- `知识博主研究-我的行动指南.md`（可直接借鉴清单 + 条件差距 + 适配建议 + 行动路线图）

### 第六步：写入检索规则，日常使用

最后一步，在你的知识库 `CLAUDE.md` 中写入以下检索规则（复制并替换路径和博主名）：

```
## 知识库检索规则

当用户提问涉及【目标博主名】时：
1. 先用 Grep 搜索知识库目录下的 *-深度汇总.md 文件
2. 命中后读取对应深度汇总文件的相关章节
3. 如果深度汇总信息不足，按文件索引追溯到单条蒸馏原文
4. 回答时标注出处（文件名 + 日期）
```

写入后，以后你问"他怎么看自学？"，AI 会自动搜索深度汇总文件，找到相关内容直接回答。

## 技术要点

- **SRT 格式是核心**：纯文本、无编码负担，AI 能直接读。导出工具几乎都支持
- **MP4 时间标注日期**：MP4 修改时间比文件名日期更可靠
- **每批 ≤30 条**：超过这个数，上下文窗口可能截断
- **模板冻结后再批量开工**：中途改模板会导致前后格式不统一
- **"我的批注"留空**：不预填，只写有触动的
- **分层检索架构**：深度汇总（日常用）→ 单条蒸馏原文（深入时按索引进）→ 跨分类综合（全局问题时用）

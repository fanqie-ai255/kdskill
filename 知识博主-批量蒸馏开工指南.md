# 知识博主-批量蒸馏开工指南

---

## 开工指令（每次新对话窗复制这段发送。⚠️ 请先将路径替换为你自己的目录）

```
读取 你的路径/知识博主-批量蒸馏开工指南.md，执行任务。
批次：【第X轮，完整文件夹路径，如"学习方法/阅读方法"】
```

**注意**：每次只蒸馏**一个子文件夹**，不要一次跨多个文件夹。AI 必须先列出该文件夹下所有 SRT 文件，确认总数，再逐条处理。

---

## AI 读取后执行的任务

### 第一步：确认范围（必须执行）

在开始蒸馏之前，你必须：

1. 列出指定文件夹下的**所有 .srt 文件**并报告数量
2. 如果该文件夹下有子文件夹，递归列出所有 .srt 文件
3. 向用户确认："本批共 X 条 SRT，我将逐一蒸馏，预计产出 X 份 .md 文件。确认开始吗？"
4. 用户回复确认后再开始

### 第二步：读取必要文件

1. **蒸馏模板**：与本指南同目录下的 `知识博主-单条视频蒸馏模板.md`
2. **本批 SRT 文件**：用户指定的文件夹路径

### 第三步：逐条蒸馏

对每条 SRT：

1. 读取文件内容
2. 去除 SRT 时间戳和序号，提取纯口播文字
3. 按模板七个栏目输出分析（一至六必须填写，七留空），其中「三、视频结构拆解」必须完整填写钩子和金句资产化字段
4. 从文件名末尾的 8 位数字提取日期（YYYYMMDD 格式）
5. 如无日期，取文件的 MP4 修改时间
6. 保存 .md 文件，命名规则：`【核心观点简化，≤20 字】.md`
7. 保存到同一文件夹下（与 SRT 同级）
8. **每处理完一条必须报进度**："已完成 X/Y：【文件名核心】

### 第四步：完成报告

全部处理完毕后输出：

```
本轮蒸馏完成
- 文件夹：XX
- 应处理：XX 条
- 实际处理：XX 条
- 产出 MD 文件：XX 个
- 可进入钩子主库（S/A）：XX 条
- 可进入金句主库（S/A）：XX 条
- 需要降级或复核的钩子/金句：列出文件名和原因
- 遗漏/失败：列出未处理的文件名
```

---

## 输出质量要求

- **道法术提炼**：每条视频最多各 2-3 个要点，宁缺毋滥。讲"道"的视频法和术可以写"本条未涉及"，不要硬凑
- **视频结构拆解**：只基于字幕能判断的内容，不要猜测画面、剪辑、配乐
- **开头钩子**：不要只写旧分类，必须同时给出心理触发器、开头动作、开头风格、可迁移模板、风险和等级。弱钩子要降级，不要为了凑数标 A。
- **金句清单**：优先选择可独立传播、可改写、能服务结构链的句子；每条必须标注功能、句式模式、可用位置、可迁移模板、风险和等级。
- **为什么有效**：重点分析身份叙事和口吻。情绪钩子和记忆点如果无法从字幕判断，写"需结合视频画面判断"
- **补充标签**：内容矩阵定位必须四选一，不能留空
- **与其他视频的关联**：首轮蒸馏无参考时简写"待汇总时补充"

## 批量简化规则

同一文件夹下内容高度重复的视频，第 2 条起可简化：
- 一、原文：正常
- 二、道法术：只写新增要点，相同写"同【前一条文件名】"
- 三、视频结构：只写钩子类型和主体结构，其余"同前"
- 钩子和金句资产化字段不能写"同前"；每条视频至少要独立判断开头钩子和金句质量，如果质量差要标 C/D 或写"本条无可入库金句"
- 四、为什么有效：只写不同部分
- 五至七：正常

## 常见遗漏原因及避免方法

| 原因 | 表现 | 避免方法 |
|------|------|---------|
| 只读了文件夹第一层，没递归子目录 | SRT 数量与实际不符 | 用 `ls -R` 或 `glob **/*.srt` 递归列出 |
| 中文路径导致 ls/glob 失败 | 部分文件夹被跳过 | 用 Python `os.walk` 遍历，不依赖 shell ls |
| 一次性处理太多文件，上下文截断 | 尾部文件被遗漏 | 每 5 条检查一次进度，确保无遗漏 |
| 文件列表未去重 | 同一文件被分析两次 | 处理前打印完整文件清单，完成后交叉验证 |

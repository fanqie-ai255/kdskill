# 知识博主-综合汇总第一阶段：分类深度汇总

> **目标**：产出分类深度汇总文件（每个分类一份）。这是你日后日常提问时 AI 最先检索的入口层。
> **前提**：所有分批单条蒸馏已完成。

---

## 开工指令（复制到新对话窗）

```
【综合汇总-第一阶段：分类深度汇总】

1. 读取蒸馏模板（与本指南同目录下的 `知识博主-单条视频蒸馏模板.md`），了解 7 个栏目的含义。
2. 遍历你的蒸馏产物目录下所有分类文件夹及其子文件夹。
3. 对每个分类，只读 .md 蒸馏文件的"道法术提炼 + 视频结构拆解中的钩子金句字段 + 补充标签"精华部分（约 300-500 字/条），不要读全文。
4. 为每个分类生成一份 `【分类名】-深度汇总.md`，存入分类文件夹根目录。
5. 每完成一个分类报进度："已完成 X/总数：【分类名】-深度汇总.md（XX 字）"。
6. 全部完成后输出简报：各分类各处理了多少条、产出了多少字、有无数据不足的分类。
```

**前置条件**：所有分批单条蒸馏已完成。

## AI 执行的任务

### 【角色】

你是一位知识体系架构师，擅长从大量碎片化分析中提炼完整的认知框架。

### 【任务】

读取 `<知识库根目录>/目标博主/` 下所有 `.md` 蒸馏文件（不含 .srt），为每个分类各生成一份**深度汇总文件**。

### 【读取策略】

不是读所有视频的全文。每个文件只提取以下字段：
- 标题（文件名）
- 道法术提炼（二栏的全部内容）
- 开头钩子（三栏第 1 项，重点取心理触发器、开头动作、开头风格、等级、风险）
- 为什么有效（四栏的 4 个子项摘要）
- 补充标签（五栏的内容矩阵定位 + 选题策略）
- 金句清单（三栏第 3 项，重点取功能、句式模式、可用位置、等级）

提取方法：直接读取每个 .md 蒸馏文件，截取 `## 二、道法术提炼` 到 `## 四、` 之间的内容，再补充 `## 五、补充标签` 的摘要。每个文件只取约 300-500 字精华，不要读全文。

### 【产出物】

为每个分类生成一份 `【分类名】-深度汇总.md`，存入 `<知识库根目录>/目标博主/【分类名】-深度汇总.md`。

每份文件结构：

### 【各分类专属维度表（示例）】

> ⚠️ 以下为 9 分类示例。实际使用时，请根据你的分类自行设计专属分析维度。

AI 处理每个分类时，必须在深度汇总的「六、分类专属分析」中填写该分类对应的维度：

| 分类 | 通用维度之外，这些是本分类独有的 |
|------|-------------------------------|
| **01学习方法** | ① **方法适用阶段**：入门/进阶/高阶，各阶段的视频分布比例；② **与我当前学习路径的映射**：本分类的哪些方法可以直接对应到我正在学的 AI 学习路径（阶段一至四）；③ **可操作复现度**：这个方法读者看完视频能直接照做吗？（高/中/低），哪些方法需要前置条件 |
| **02赚钱创业** | ① **商业模式可行性评估**：他讲的每种赚钱方法，不同起步基础和资源条件的人能做吗？（能直接做 / 需要学习前置技能 / 需要资金投入）；② **成本门槛分级**：零成本 / 少量资金（<1000元）/ 需要专业积累 / 需要团队；③ **和用户当前条件的适配度**：用户当前条件，能直接尝试的有哪些 |
| **03认知思维** | ① **认知升级路径**：从本分类的视频中，能否归纳出一条"认知升级的阶梯"（从最基础到最高阶的思维模型排列）；② **常见认知偏差对照**：他反复提到的"大多数人以为 X，其实 Y"，整理成对照表 |
| **04自媒体创作** | ① **难度分级**：新手可做 / 有基础后可做 / 需要长期积累；② **和用户内容方向的直接关联**：本分类中哪些技巧直接适用于口播视频（不需要露脸/不需要团队）；③ **内容类型与发布频率建议**：从他的实践中提炼的发布策略 |
| **05行动习惯** | ① **为什么人知道但做不到**：本分类视频中给出的"从知道到做到"的机制是什么；② **触发条件**：他说的每种习惯养成方法，是在什么场景下触发的（早晨/遇到困境/日常/复盘时）；③ **最低启动门槛**：他建议的"最小行动"是什么，用户能在可投入时间内完成的 |

> 以下分类的视频数量较少（<30 条），专属维度宜精简：

| **06投资理财** | ① **适用人群**：他讲的投资方法适合什么人（有积蓄的/刚工作的/初学者）；② **和用户当前阶段的关系**：这些内容用户当前阶段需要关注还是先知道就行 |
| **07人际社交** | ① **社交策略类型**：他讲的是"多交朋友"还是"减少无效社交"还是"价值交换"；② **可立即用的**：哪条建议我今天就能用 |
| **08个人成长** | ① **成长阶段模型**：如果他把个人成长分成几个阶段，分别是什么；② **他反复讲的那几个故事**：哪些个人经历故事在本分类中反复出现 |
| **09其他** | ① **是否可归入前 8 个分类**：如果可以，建议该条归入哪个分类；② **有没有值得单独拎出来的新主题** |

### 【质量要求】

- "出处文件"一栏是给 AI 看的——你回答用户问题时，如果用户对某条不满意，按这个文件名去读原文
- "经典原文"必须是直接从蒸馏文件里引用的，不要改写
- 频次标注必须真实统计，不要估计
- 每份文件 5000-18000 字（含分类专属分析后允许略超原来的 15000 上限）
- **分类专属分析的每个维度必须填写，不能留空。如果数据不足以支撑某个维度，写"本分类数据不足，建议补充 X 条视频"后给出方向性判断**

### 【执行步骤】

1. 遍历所有分类文件夹，统计每个分类的 .md 蒸馏文件数量，报告给用户
2. 确认后，按分类逐个处理：提取 → 归并 → 输出深度汇总
3. 每完成一个分类，报告进度："已完成 X/总数：【分类名】-深度汇总.md（XX 字）"
4. 全部完成后输出简报
